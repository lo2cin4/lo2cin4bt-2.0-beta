策略建立層級

這頁解釋 lo2cin4bt 如何把一句策略想法，變成可檢查、可回測、可由瀏覽器前端（frontend）顯示的本機研究流程。

安全邊界

普通策略開發只應產生 workspace/ 內的資料、特徵合約、自訂指標、策略配置檔（config）和筆記。核心引擎（engine）只有在明確提出引擎功能請求（engine feature request）時才修改。

第一層：用戶 / AI 可寫的設定

workspace/runs/

可執行回測設定，使用 schema_version: "strategy_run"。

workspace/wfa/

滾動驗證（WFA / walk-forward analysis）設定，使用 schema_version: "wfa_run"，並指向一份回測配置檔（run config）。

workspace/features/

外部資料合約。每個特徵（feature）必須說明資料在現實中何時才知道，避免回測偷看未來。

workspace/indicators/extensions/

自訂指標擴充（extension）。需要 manifest.json 和 indicator.py，並通過檢查器（doctor）；要真正跑回測，還需要執行層（runtime）支援調動該指標。

第二層：策略設定欄位

新的公開寫作介面（authoring surface）使用以下欄位：

  • computed_fields[]：計算值，例如簡單移動平均（SMA）、相對強弱指標（RSI）、動量（momentum）、平均真實波幅（ATR）。
  • signals：入場 / 出場訊號。
  • selection：多資產排序與合資格條件（eligibility）。
  • allocation：權重、前 N 名選取（top-N）、持倉上限。
  • rebalance：定期、事件或訊號再平衡。
  • fill_model：入場價、出場價、延遲 K 線數（delay bars）、成本、滑價。

新的配置檔（config）不應使用頂層（top-level）features、頂層 indicators 或 execution 作為公開寫法。

第三層：檢查與編譯

1 結構驗證（schema validation）
2 工作區檢查器（workspace doctor）
3 指標檢查器（indicator doctor）
4 支援範圍檢查（support checker）
5 執行層派發（runtime dispatch）
6 前端顯示資料包（frontend display payload）

偷看未來偏誤閘門（Look-Forward Bias Gate）

外部資料必須寫明：

{
  "data_availability": {
    "observed_at": "bar_close",
    "usable_from": "next_bar_open",
    "point_in_time": true,
    "revision_policy": "point_in_time"
  }
}

如果資料在收盤後才知道，就不能用來做同一根 K 線開盤交易。若資料會修訂歷史（revision_policy: "revised_history"），它只代表可用於研究示範或需要再審查，不能說成已證明逐時點無偏誤。正式公開結論前必須留下量化審查（QuantReview）紀錄。

不支援就要停下來

如果策略需要未支援的引擎（engine）行為，AI 應回覆 unsupported_needs_new_building_block，而不是用人造價格曲線、檔名猜測、或自然語言假裝已支援。

維護者檢查

公開前維護者應跑：

python scripts/workspace_doctor.py
python scripts/indicator_doctor.py workspace/indicators/extensions
python scripts/project_consistency_audit.py
python scripts/frontend_payload_parity_audit.py
python scripts/quality_gate.py --quick
