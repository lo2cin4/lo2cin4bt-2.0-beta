# AI-Readable Result Output Contract

Status: implemented foundation.

Every completed or partial app run writes:

```text
outputs/app/ai_review/{run_id}/ai_review_pack.json
```

The pack is generated by `app.api.payloads.AppPayloadService` after
the normal app payloads are produced.

## Purpose

The file is the stable handoff between lo2cin4bt and a future AI review skill.
It is not a prose report and does not make investment conclusions by itself.
It is a machine-readable evidence bundle.

## Auto-Inclusion Rule

The pack must not depend on a hand-maintained list of metric names.

It includes:

- the run registry entry
- stage status
- full artifact manifest
- every JSON file in `outputs/app/chart_payloads/{run_id}/`
- every direct JSON snapshot in `outputs/app/run_snapshots/{run_id}/`
- artifact index rows for every manifest artifact
- table profiles for ready parquet/csv artifacts, grouped by artifact type
- a numeric field catalog discovered recursively from embedded payloads and
  artifact profiles

This means future metrics are automatically available to AI when they are added
to app payload JSON or artifact table columns.

## Future AI Review Skill Boundary

A future review skill should read the pack first, then:

1. verify run status, data health, and artifact readiness
2. summarize strategy intent and execution assumptions
3. compare headline performance, benchmark, drawdown, risk, and costs
4. review parameter matrix / WFA / stat evidence when present
5. identify missing artifacts, unsupported cards, warnings, and insufficient
   evidence
6. label anything not found in the pack as `not generated` or `not available`

The skill must not treat absent fields as zero and must not invent metrics that
are not present in the pack or referenced artifacts.
