Module 03

StrategyRun Config

strategy run config 是 Lo2cin4BT 目前偏好的回測設定格式。它的目標很簡單： 用同一份 config 描述資料、資產、策略邏輯、交易成本、風險限制、參數搜尋和輸出，讓單資產、 日曆事件、多資產投資組合都能走同一個執行合約。

核心規則： config 只描述策略與工作流；真正計算由 dataloader、factorhandler、backtester、WFA、statanalyser 各自負責。

Config 到報告的路徑

flowchart LR
  User["用戶策略描述"] --> AI["AI 按 Manual 判斷<br/>支援 / 要追問 / 不支援"]
  AI --> Config["strategy run config"]
  Config --> Data["data + universe"]
  Config --> Logic["computed_fields + signals + selection"]
  Config --> Portfolio["allocation + rebalance + risk"]
  Data --> Runner["Unified Backtest Runner"]
  Logic --> Runner
  Portfolio --> Runner
  Runner --> Artifacts["managed artifacts<br/>parquet / json payload"]
  Artifacts --> Report["Backtests 回測 / Metrics 概覽"]
  Artifacts --> Matrix["Parameter Matrix 參數矩陣"]
  Artifacts --> WFA["WFA 滾動驗證"]

讀 config 時不用一次背完所有欄位；先分成資料、策略邏輯、投資組合、驗證輸出四組，會易讀很多。

Config 的主要區塊

區塊 用途 常見可調位置 前端會用到哪裡
platform 標記策略模式、工作流、run type。 strategy_mode_id、workflow_id、label。 Run Center 執行中心、Metrics file selection、Strategy Rules。
data 指定 provider、頻率、calendar、timezone、benchmark。 yfinance、binance、coinbase、file、benchmark。 Data Health（資料健康）、Benchmark、Effective Start（有效起始日）。
universe 策略可交易資產池。 單一 symbol 或多資產列表。 Strategy Rules、Portfolio Context、Holdings。
computed_fields / signals 計算 SMA、momentum、RSI、MACD、calendar mask 或 入場 / 出場 signal。 indicator period、price field、cross rule、交易日曆規則。 Strategy Rules、Parameter Matrix 參數矩陣、WFA domain。
selection 多資產策略用來決定哪些資產入選。 eligible、rank_by、top_n、rank order。 Allocation Timeline、Target Holdings、Asset Contribution。
allocation 把 signal 或 selection 轉成 目標權重。 equal weight、fixed weights、position limit、cash policy。 Portfolio Equity、Scheduled Rebalances、Allocation Changes。
rebalance 決定何時檢查或調整 portfolio。 every session、month start、year start、event date、signal change。 Active Rebalances、Checkpoints、Turnover。
fill_model 定義成交時間、價格、交易成本和滑點。 open/close、next bar、transaction cost、slippage。 Trade Return、Cost Drag、Net-vs-gross future analysis。
risk 可選的風險閘門。 max positions、max daily loss、max order size、max MDD。 Risk Gate Audit。
parameter_domains 描述可搜尋參數範圍。 MA period、weekday、ordinal、lookback、allocation profile。 Parameter Matrix 參數矩陣、WFA optimization。

Mode 與 Workflow 要分開理解

strategy_mode_id

描述策略形態，例如單資產 signal、多資產 portfolio、日曆事件、動態配置。

  • single_asset_signal
  • multi_factor_entry_exit_roles
  • calendar_event_session
  • multi_asset_portfolio

multi_asset_trigger_selection 和 dynamic_allocation_rules 目前是 planned / reserved mode。AI 不應把它們寫成可執行 config，除非 schema、support checker、runtime、oracle test 和 QuantReview 都確認支援。

workflow_id

描述這次要產生什麼結果，例如單次回測、參數矩陣、WFA、滾動驗證。

  • single_backtest
  • parameter_matrix
  • walk_forward_analysis
  • rolling_validation
  • statanalyser

產生 config 時的處理原則

  • 用戶描述可以很自由；AI 將會轉換成本專案已支援的 config 合約。
  • 如果某策略無法用現有合約表達，AI 將會明確說明「目前 Lo2cin4BT 不支援」，不會硬寫成看似可跑的 config。
  • 有參數範圍才使用 parameter_matrix 或 WFA optimization；固定策略會使用 single backtest 或滾動驗證。
  • benchmark、provider、calendar、timezone 會以市場口徑檢查；AI 不會在未說明下猜測市場。
  • 成本、滑點、fill_model timing 要顯示在 Strategy Rules，方便用戶審核。
  • 使用外部資料時，feature contract 必須寫 data_availability，包括 observed_at、usable_from、point_in_time 和 revision_policy。
