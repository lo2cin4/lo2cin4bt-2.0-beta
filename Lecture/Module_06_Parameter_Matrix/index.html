Parameter Matrix 參數矩陣

Parameter Matrix 參數矩陣是「候選參數研究」。它把 parameter_domains 展開成多個候選， 每個候選都用同一段樣本回測，再用 metrics、heatmap、cluster 與 candidate review 協助你判斷是否存在穩定區域。 這不是 WFA，也不是把整段樣本的最佳解直接視為 OOS 證據。

Parameter Matrix 與 WFA 的差異

flowchart TB
  Domain["parameter_domains"] --> Matrix["Parameter Matrix
整段樣本測候選"]
  Matrix --> Rank["ranking / heatmap / cluster"]
  Rank --> Review["候選檢視與穩定區域"]

  Domain --> WFA["Walk-Forward Analysis"]
  WFA --> Split["切分 IS / OOS window"]
  Split --> IS["每段 IS 尋優"]
  IS --> Pick["每段選出 rank-1 optimum"]
  Pick --> OOS["只把該 optimum 放到 OOS"]
  OOS --> Robust["穩健性結論"]

Matrix 回答「整段資料有沒有穩定候選」；WFA 回答「每段 IS 找出的最佳參數，到下一段 OOS 是否仍然可靠」。

Parameter Matrix

  • 一次展開所有候選參數。
  • 產生 ranking、cluster、heatmap、candidate review。
  • 適合尋找穩定區域，而不是只追單一最高分候選。
  • 不等於 OOS 穩健性證據。

WFA 滾動驗證

  • 每個 IS window 重新尋找當段最佳參數。
  • OOS 只測該 window 選出的最佳參數。
  • 適合檢查參數是否穩定、OOS 是否仍然有效。
  • 候選 grid 可以作 diagnostics，但不能直接當作 pass/fail evidence。

哪些內容適合放進 Matrix

技術指標週期SMA、momentum、RSI、MACD 等 period。
Calendar 參數weekday、ordinal、月份、事件日期規則。
配置候選多組 allocation profiles、top_n、position limit。
交易與風控cost、slippage、risk gate threshold。

No-domain 的正確解讀

固定策略如果沒有任何可調參數，例如固定 30/10/20/20/20 配置，系統應回傳 availability = "no_parameter_domain"，前端會提示「沒有參數矩陣」並引導你到 Backtests 回測。 這不是錯誤，而是表示目前 config 沒有可展開的候選參數。

前端狀態 代表什麼 下一步
有 heatmap / candidate review 策略有可調參數，Matrix 已生成候選。 挑穩定區域，再用 WFA 或 Rolling Validation 驗證。
No parameter domain 固定策略無可調參數。 看 Backtests 回測，或把配置改成多個 allocation profile。
Unable to load payload artifact 或 API payload 未能載入。 檢查 managed artifacts 與 API payload。
