Module 02

資料來源與安裝要求

Lo2cin4BT 的資料來源統一經過 dataloader/。Backtester、WFA、前端不會直接呼叫 broker、 exchange 或外部資料 API。這個邊界讓策略、回測與前端報告可以共用同一套資料健康檢查。

資料如何進入回測

flowchart LR
  Request["config data section
provider / symbols / benchmark"] --> Adapter["dataloader adapter"]
  Adapter --> Standard["標準 OHLCV
date / open / high / low / close / volume"]
  Standard --> Health["data health
effective start / missing assets"]
  Health --> Cache["cache / managed artifacts"]
  Cache --> Engine["backtester / WFA"]
  Health --> Rules["Strategy Rules
前端可讀摘要"]

不同資料來源的差異留在 adapter；引擎只處理標準化後的資料與資料健康報告。

正式資料來源規則

Provider 用途 資產代號例子 需要什麼 常見錯誤
yfinance 公開每日 ETF / 股票資料。 QQQ、SPY、VOO requirements.txt 內的 yfinance / curl_cffi。 資產代號不存在、Yahoo 缺資料、比較基準與策略市場不一致。
binance crypto spot kline。 BTCUSDT、ETHUSDT python-binance、網絡、正確 時間週期。 不會在未說明下改用 Yahoo benchmark；Binance symbol 不使用連字號。
coinbase Coinbase Exchange 公開 K 線資料。 BTC-USD、ETH-USD dataloader/coinbase_loader.py、網絡、Coinbase product 格式。 Coinbase 用 BASE-QUOTE 格式，不使用 Binance 的 BTCUSDT 寫法。
file 本地 CSV / parquet。 本地路徑與 資產代號對應 日期欄、OHLCV 欄位、timezone / calendar 定義。 欄位名不一致、日期不是交易日、缺 benchmark、adjustment 未說明。
futu FUTU OpenD 轉接器框架。 依 Futu market prefix futu-api、Futu OpenD、host/port、行情權限。 套件裝好不等於 OpenD 已開，也不等於有行情權限。
ibkr IBKR TWS / IB Gateway 轉接器框架。 依 IBKR contract / 資產代號對應 ib-insync、TWS/Gateway、API enabled、port、market-data entitlements。 Gateway 未開、API port 錯、market data entitlement 不足。

Data config 要寫什麼

{
  "data": {
    "provider": "binance",
    "frequency": "1D",
    "calendar": "CRYPTO_24_7",
    "timezone": "UTC",
    "benchmark": "BTCUSDT"
  },
  "universe": {
    "symbols": ["BTCUSDT", "ETHUSDT"]
  }
}

Benchmark 原則

  • 單資產策略可由同一資料來源的買入持有 推斷，但仍建議明確寫入。
  • 多資產 portfolio 應明確指定 benchmark，例如 SPY、BTCUSDT 或同市場代表資產。
  • crypto 資料來源不會在未說明下切到 Yahoo ticker；如果要跨資料來源 benchmark，AI 將會先向用戶說明。
  • 若比較基準缺資料，報告會影響 Benchmark Return、Excess Return、Benchmark Correlation。

Data Health（資料健康） 解讀

Effective Start（有效起始日）

所有必要資產、benchmark、features 可共同開始的日期。多資產會被最晚可用資產拖後。

Loaded Assets（已載入資產）

實際成功載入的資產。應與 universe 對照。

Missing Assets（缺失資產）

要求但未成功載入的資產。若存在，策略結果通常需要先檢查資料缺口。

Data Provider Error（資料來源錯誤）

例如 端點、閘道程式、權限、資產代號或時間週期 問題。這不是策略邏輯錯誤。
