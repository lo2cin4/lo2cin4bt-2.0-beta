Module 12

Validation Tools 驗收工具

這一章給進階用戶與維護者閱讀，用來理解測試、覆蓋率、安全掃描、golden regression 與量化有效性 gate。新手只需要知道如何做健康檢查，已放在 Module 00。

進階：兩個總入口

情況指令用途
日常修改後 python scripts/quality_gate.py --quick 跑快速品質檢查，適合大部分本機改動。
公開或重要版本前 python scripts/quality_gate.py --full 跑完整驗收，包括覆蓋率、安全、前端建置、Rust 與更多測試。

進階：背後包含的工具

工具檢查甚麼為何存在
pytestPython 測試、oracle 測試、golden regression防止核心行為被改壞。

進階：golden regression 是甚麼

Golden regression 是「固定答案測試」。系統用固定本機假資料重跑內建策略範例，然後對比預先鎖住的結果，例如 final equity、交易次數、參數組合數和成本狀態。

如果日後改 engine、config 或前端 payload 時，回測結果偷偷變了，這個測試會失敗。它不是用來證明策略會賺錢，只是用來證明軟件計算沒有無聲漂移。

  • 均線策略：BTCUSDT dual MA、QQQ SMA cross。
  • 日曆效應：每月第 N 個星期幾 same-session。
  • 特定日子調倉：年度固定資產配置再平衡。
  • Momentum：VOO / GLD momentum rotation。

進階：量化有效性 gate

  • open / close 與 delay bars 必須清楚定義，避免訊號和成交時間混亂。
  • 成本與滑價必須真的進入計算，不能只顯示在文字上。
  • 外部資料必須說明資料在現實中何時才知道，避免回測偷看未來。
  • benchmark 必須清楚，否則策略表現沒有合理參照。
  • Parameter Matrix 必須真的是多組參數；單一參數應使用單次回測。
  • WFA 只是驗證流程，不是盈利保證。

Workspace-only 原則

普通策略、資料、indicator、WFA 設定，應在 workspace/ 內完成。自訂 indicator 屬於程式碼，需要 doctor 檢查和 runtime 支援；如果未支援，AI 應停下回報缺少功能。只有當現有 engine 不支援新能力時，才應改 backtester/、app/、autorunner/ 或 scripts/，並交由進階維護流程驗收。
