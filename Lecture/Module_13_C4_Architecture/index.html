C4 Architecture 架構圖

C4 是一種由外至內理解系統的方法。先看誰使用系統，再看系統由哪些大模組組成， 最後才看每個模組內部如何協作。

Context：誰在用這個系統

flowchart LR
  User["用戶 / 研究員"] --> AI["AI 助手"]
  AI --> Repo["lo2cin4bt 開源框架"]
  Repo --> Browser["本機瀏覽器前端"]
  Repo --> Data["行情 / 本機資料"]
  Repo --> Reports["回測、WFA、AI 證據包"]

Component：策略從想法到結果

策略想法 用戶描述想測試的交易規則
StrategyBuilder skill 把想法整理成可檢查的設定
schema / contract 限制 config 欄位與格式
backtest engine 計算交易、成本與持倉
runtime dispatch 選擇正確 engine path
doctor gates 不通過就打回重做
display payload 後端提供正式顯示欄位
frontend 只顯示 payload，不自行猜策略

Code-level gate：防止說法與實際不一致

  • 前端只能顯示後端 payload 的正式欄位，不應自行猜策略語義。
  • schema、validator、runtime、docs、lecture 必須描述同一套欄位。
  • 普通策略和 indicator 開發應只在 workspace 內完成；自訂 indicator 通過 doctor 不代表一定可跑，還要有 runtime dispatch 支援。
  • 需要改核心時，才升級為 engine feature request。
  • Quant gate 檢查 open / close、delay bars、成本、滑價、benchmark、look-ahead、WFA 和 parameter matrix。
