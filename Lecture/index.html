先理解整條工作流，再進入單一章節；這樣讀報告時不會只盯着一個績效數字。

AI 使用原則： 先請你的 AI 閱讀 docs/ai/AI_MANUAL_SKILL.md， 再讀 README、refactor 文件、相關 config、tests 與輸出檔案。 若答案超出 repo 內證據，AI 將會標示為 AI 推論、外部建議或未來設計。

學習路線

1. 新手啟動

Module 00 到 04：如何準備 AI Manual、安裝依賴、啟動平台、完成第一次回測。

2. 策略設計

Module 03 到 06：理解 strategy run 設定、自然語言策略如何轉換成 config、如何做 Parameter Matrix 參數矩陣。

3. 報告閱讀

Module 07 到 09：逐項閱讀 Metrics 概覽、Backtests 回測、WFA 滾動驗證、Risk Gate、Portfolio Invariant 與各類數字。

4. 擴充與未來

Module 10 說明資料來源擴充；Module 11 標註 FactorHandler / Factor Analysis 因子分析的尚未開發功能邊界；Module 12 到 13 說明驗收工具與 C4 架構圖。

用戶可以學會什麼

1

用自然語言描述策略

例如「BTC 和 ETH 各自收盤價高於 50 日均線才持有」或「每個月第一個星期一開市交易QQQ，收市平倉」。

2

確認策略是否支援

AI 將會依 Manual 回報 supported、needs_clarification 或 unsupported，不會編造功能。

3

生成與執行 config

用 strategy run config 表達 provider、universe、features、signals、allocation、rebalance、fill model、risk 與 parameter domains。

4

閱讀報告與驗證穩健性

從 Metrics 概覽到 Backtests 回測，再到 Parameter Matrix 參數矩陣與 WFA 滾動驗證，逐步確認績效、風險、成本與資料健康。
