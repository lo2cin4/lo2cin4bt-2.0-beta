Module 04

回測入門

Lo2cin4BT 的回測不是單純畫一條資金曲線。系統會先讀取策略設定，載入資料， 產生 features、signals 或 target weights，再用投資組合記帳核心逐步計算現金、 持倉、市值、費用、turnover、equity 與 benchmark。

選 config 載入資料 產生訊號 / 權重 逐步記帳 輸出 artifacts 前端閱讀

可一次計算的 signal、indicator、ranking 會先向量化；涉及現金、持倉、費用和風控狀態的部分，會逐步記帳。

單資產如何進入統一引擎

單資產策略會被視為「只有一個資產的投資組合」。例如 QQQ 均線策略： 入場訊號把 QQQ target weight 設為 100%，離場訊號把 target weight 設為 0%。 這樣它可以與多資產投資組合共用同一套 equity、fee、turnover、risk gate 與 invariant 檢查。

實務操作順序

  1. 啟動平台 執行 python main.py，打開 http://127.0.0.1:2424/。
  2. 選擇 Run Center 執行中心 config 從 workspace/runs/ 選 backtest config；有參數範圍時會是 Parameter Matrix 參數矩陣，無參數時是 Single Backtest 單次回測。
  3. 閱讀 Metrics 概覽 先看 equity、benchmark、Total Return、CAGR、Sharpe、MDD、Calmar。
  4. 進入 Backtests 回測 檢查 Strategy Table、Selected Strategy Summary、Portfolio Context、Risk Gate Audit、Allocation Changes。

回測前檢查

Config schema使用 strategy_run，或可被 normalizer 轉換。
資料完整性確認資產、benchmark、頻率、有效起始日都能載入。
Parameter domain有可調參數才會生成 Parameter Matrix 參數矩陣。
策略類型固定配置策略應看 Rolling Validation，不應誤稱 WFA optimization。
