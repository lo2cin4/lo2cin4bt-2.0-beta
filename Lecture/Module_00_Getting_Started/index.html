Module 00

開始使用與 AI Manual

Lo2cin4BT 2.0 是以瀏覽器為主要入口的量化研究與回測 workspace。 使用者以結構化 config 描述策略，再執行 單資產、多資產投資組合、 Parameter Matrix 參數矩陣、WFA 滾動驗證。

準備 AI Manual

開源使用者最重要的第一步，是先準備好 AI 要讀的正確規則，而不是憑一般量化常識亂補功能。

  1. 把 AI 工作目錄指到 repo 根目錄，例如 lo2cin4bt。
  2. 第一個讀 skills/lo2cin4bt/SKILL.md。
  3. 再按需要讀 skills/lo2cin4bt/references/、docs/ai/AI_MANUAL_SKILL.md、相關 config、tests 與輸出檔案。
  4. 要求 AI 區分 repo 已實作、尚未開發階段、外部建議或 AI 推論。
  5. 若缺 universe、provider、frequency、calendar、benchmark、入場 / 出場、allocation、risk gate 或 parameter domain，AI 將會先列出缺口，不會編造。
請先閱讀 skills/lo2cin4bt/SKILL.md。
任何 repo 內沒有證據的內容，請標示為 AI 推論、外部建議或未來設計。
如果我描述的策略目前 Lo2cin4BT 做不到，請直接說明缺口，不要強行寫 config。

安裝與啟動

以下命令是 repo 內建 的基本路線。若環境有不同 Python / Node 管理方式，需按本機情況調整。

cd lo2cin4bt
python -m pip install -r requirements.txt
cd plotter/web
npm ci
npm run build
cd ../..
python main.py

打開平台

啟動後打開 http://127.0.0.1:2424/。Run Center 執行中心會讀取 workspace/runs/ 與 workspace/wfa/。

外部 gateway

FUTU / IBKR 需要本機 套件、閘道程式、host/port 與資料權限。這不是純 config 能解決的問題。

第一次使用順序

1

Run Center 執行中心

先選一份已存在的 Backtests 回測 config，不要一開始就寫全新策略。

2

Metrics 概覽

確認 Total Return、CAGR、Sharpe、MDD、Calmar、Benchmark 比較基準與 Data Health（資料健康）。

3

Backtests 回測

看 equity、allocation changes、target holdings、asset contribution、risk gate audit 與 invariant。

4

Parameter Matrix 參數矩陣 / WFA 滾動驗證

有參數範圍才看 Parameter Matrix 參數矩陣；要驗證穩健性再看 WFA 滾動驗證。

出錯時先做健康檢查

平常研究策略時，你主要在 workspace/ 內工作。當前端看不到結果、策略跑不出來，或者 AI 改完檔案後想確認沒有弄壞東西，就請 AI 跑一次健康檢查。

可以直接複製給 AI：
請在 lo2cin4bt 專案內跑一次健康檢查：python scripts/quality_gate.py --quick。如果有錯，請用簡單語言告訴我哪一步失敗、代表甚麼、下一步如何修。

前端入口

  • Run Center 執行中心：選擇 Factor Analysis 因子分析、Backtests 回測、Walk-Forward config。
  • Metrics 概覽：總覽 Total Return、CAGR、Sharpe、MDD、Calmar。
  • Parameter Matrix 參數矩陣：檢查候選、ranking、no-domain。
  • Backtests 回測：閱讀 portfolio evidence、risk gate audit、trade / 再平衡 / holdings。
  • WFA 滾動驗證：閱讀 selected optimum OOS window、滾動驗證、portfolio evidence。
