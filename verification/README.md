# lo2cin4bt Truth Validation Workspace

This folder contains the repeatable truth-validation harness for comparing
legacy outputs, candidate outputs, and summarized diffs.

## Source-Control Policy

- Keep `fixtures/`, `scripts/`, benchmark definitions, and this README in Git.
- Keep `baseline_old/`, `candidate_new/`, and `results/` as directory anchors only
  with `.gitkeep` files.
- Do not commit generated parquet/json/md files under those run-output folders.
  They often contain machine-local absolute paths and should be regenerated by
  the scripts when needed.

## Layout

- `fixtures/`: compact inputs and manifests used by the validation scripts.
- `baseline_old/`: generated legacy-output workspace.
- `candidate_new/`: generated candidate-output workspace.
- `scripts/`: fixture preparation, batch execution, comparison, and summary tools.
- `results/`: generated comparison summaries and artifacts.

## Common Commands

```powershell
python verification/scripts/prepare_truth_validation_fixtures.py
python verification/scripts/run_truth_validation_batch.py --batch-id first_round
```

The case runners default both sides to the current project root. To compare
against an archived baseline, set:

```powershell
$env:LO2CIN4BT_BASELINE_ROOT = "C:\path\to\archived-baseline-root"
$env:LO2CIN4BT_CANDIDATE_ROOT = "C:\path\to\lo2cin4bt"
```
