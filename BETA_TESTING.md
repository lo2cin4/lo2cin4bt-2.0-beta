# lo2cin4bt 內測邀請

歡迎加入 lo2cin4bt 內測！

- GitHub 下載專案：[lo2cin4/lo2cin4bt-2.0-beta](https://github.com/lo2cin4/lo2cin4bt-2.0-beta)
- Telegram：[lo2cin4group](https://t.me/lo2cin4group)
- Discord：[lo2cin4bt community](https://discord.com/invite/PhFN83Nq6w)

lo2cin4bt 是一個以 AI 輔助為核心、同時希望讓新手也能上手的開源回測框架。
這次內測希望邀請你協助試用、探索、回報問題，並提供真實使用意見，幫助我們改善文件、平台體驗、回測流程與 AI 代理協作。

## 開始方式

1. 到 GitHub 下載專案：[lo2cin4/lo2cin4bt-2.0-beta](https://github.com/lo2cin4/lo2cin4bt-2.0-beta)
2. 加入 [Telegram](https://t.me/lo2cin4group) 或 [Discord](https://discord.com/invite/PhFN83Nq6w)
3. 參考 README 完成安裝與啟動
4. 在瀏覽器打開可視化回測平台
5. 如有 AI 輔助，可直接使用 README 提供的 prompt，請 AI 代理協助你探索框架

## 新手應該能夠做到的事情

- 成功啟動 lo2cin4bt
- 在瀏覽器打開可視化回測平台
- 在平台上找到並執行內建回測策略範例
- 嘗試回測 5 個內建策略範例
- 查看回測結果、圖表、指標、持倉與交易紀錄
- 打開教學 HTML 或相關教學文件
- 使用 AI 代理 `lo2cin4bt_Teacher` 學習平台操作方式
- 使用 AI 代理 `lo2cin4bt_StrategyBuilderSubAgent` 嘗試開發策略
- 由 AI 代理 `lo2cin4bt_PM` 協調其他 AI 代理，檢查策略是否可行、是否符合框架支援範圍，以及是否存在明顯偏誤風險

## 如果你有 AI 輔助

你可以直接複製 README 中提供的 prompt，請 AI 協助你：

- 安裝與啟動專案
- 解釋各個頁面的用途
- 執行內建策略範例
- 建立新的策略配置檔
- 檢查策略是否符合目前框架能力
- 檢查是否存在前視偏誤、資料問題或不合理假設
- 解釋回測結果的含義，以及哪些內容不應過度解讀

## 如果你沒有 AI 輔助

你也可以自行嘗試：

- 依照 README 完成安裝
- 啟動本地平台
- 在瀏覽器打開可視化介面
- 執行內建策略範例
- 查看結果頁面與圖表
- 進階使用者可自行修改或撰寫配置檔
- 如遇到問題，可在群組中提出，我們會協助定位問題

## 不應該遇到或做到的事情

- 不應該需要修改 `workspace/` 以外的核心代碼才能正常使用
- AI 代理不應該能夠建立有明顯前視偏誤的策略而完全沒有警告
- 程式不應該能夠將目前尚未支援的策略邏輯偽裝成可執行配置
- 程式不應該被引導進行真實下單、實盤交易、移動資金或修改券商帳戶設定
- 不應該需要提交 API key、券商密碼、私人資料或任何敏感資訊

## 這次希望你協助評價的地方

### 1. README 是否清晰？

- 安裝步驟是否容易跟隨？
- 你是否能理解這個框架可以做到什麼？
- 有 AI / 沒有 AI 的使用方式是否說明清楚？

### 2. 第一次啟動是否順利？

- 安裝過程是否有卡住？
- 指令是否有報錯？
- 瀏覽器平台是否能成功打開？
- Windows / macOS / Linux 是否有特別問題？

### 3. 可視化平台是否有異常？

- 頁面是否能正常打開？
- 按鈕是否有無反應的情況？
- 圖表是否有空白、錯位或載入失敗？
- 結果頁面是否容易理解？

### 4. 內建策略範例是否可用？

- 是否能夠成功執行？
- 結果是否能正常顯示？
- 5 個內建策略範例是否足夠讓新手理解框架能力？

### 5. 回測結果是否有可疑或錯誤之處？

- 指標是否合理？
- benchmark 是否清楚？
- 交易紀錄、持倉與 rebalance 是否符合策略邏輯？
- 成本、slippage、drawdown 是否有清楚展示？

### 6. AI 代理體驗如何？

- `lo2cin4bt_Teacher` 是否解釋清楚？
- `lo2cin4bt_StrategyBuilderSubAgent` 是否能正確理解策略想法？
- `lo2cin4bt_PM` 是否能正確分派檢查、回測與分析任務？
- AI 是否有說錯、誇大，或引導你做不應該做的事情？

### 7. 一般回測框架應該關注的地方

- 資料來源是否清楚？
- 回測時是否有前視偏誤風險提示？
- 策略設定是否可重現？
- 結果 artifact 是否容易保存與分享？
- 前向分析 WFA / 參數矩陣 parameter matrix 是否容易理解？
- 錯誤訊息是否足夠幫助你解決問題？

### 8. Bug、報錯或卡關

- 任何安裝錯誤
- 任何執行錯誤
- 任何頁面異常
- 任何 AI 代理答非所問
- 任何 README 說明與實際操作不一致的地方

### 9. 你希望新增的功能

- 更多內建策略範例
- 更多教學內容
- 更簡單的策略建立器
- 更清楚的結果解釋
- 更多資料來源
- 更完整的錯誤提示
- 更容易分享回測報告的方式

### 10. 你認為不必要或過於複雜的功能

- 是否有頁面資訊過多？
- 是否有功能對新手來說暫時不需要？
- 是否有術語過於艱深？
- 是否有流程可以進一步簡化？
